# CLAUDE.md

## Design Context

Product strategy and the visual design system live in two root files. **Read both before any UI or product work:**

- **[PRODUCT.md](PRODUCT.md)** — strategy: register (`product`), platform (`adaptive`), users, purpose, positioning, brand personality, anti-references, design principles, accessibility.
- **[DESIGN.md](DESIGN.md)** — visual system: color tokens, typography, elevation, components, do's & don'ts. `.impeccable/design.json` is the machine-readable sidecar.

**In one line:** LamTo is the accountability layer for apartment maintenance — a Vietnamese-first resident mobile app (Flutter, iOS + Android) and a desktop-first staff web platform, bound by one evidence chain. Two visual systems, shared brand + integrity tokens; Accountability Indigo (`#2f3a8f`) ≤10% of any screen; explain before proving.

Run `/impeccable <command>` (craft · shape · critique · audit · polish · …) for design work; it reads these files first.
